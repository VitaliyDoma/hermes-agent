## 🎯 ГЛАВНАЯ ЗАДАЧА

> **Собери канонический план реализации Hermes v3 с модулями, зависимостями, приоритетами и Definition of Done.**

Hermes v3 — это полуавтономный разговорный AI-ассистент, который:
- Находит **кандидатов на обучение** (а не партнёров, конкурентов, экспертов)
- Объясняет **почему** выбран конкретный лид
- Знает актуальные **продукты и курсы Дарьи** (источник: darangel.ru)
- Пишет **первые сообщения** в нормальном тоне от лица Дарьи
- Запоминает **обратную связь** и улучшает отбор
- **Не применяет** стратегические изменения без подтверждения Дарьи
- **Экономит токены** через мультимодельный стек

## 📦 МОДУЛИ HERMES V3

### M1 — Conversational Router
**Приоритет:** 🔴 P0 (критический)  
**Зависимости:** нет  
**Слой:** Conversational Layer

**Что делает:**
- Принимает свободный текст от Дарьи
- Распознаёт интент (поиск / оценка / обратная связь / настройка / вопрос по продуктам)
- Задаёт уточняющий вопрос при неоднозначности
- Переводит в action plan для Action Layer
- Подтверждает рискованные изменения перед применением

**Модель:** Малая быстрая модель (GPT-4o-mini / Claude Haiku)

**Definition of Done:**
- [ ] Распознаёт 7 базовых интентов с точностью ≥ 90% на тест-сете из 50 примеров
- [ ] Задаёт уточнение при confidence < 0.7
- [ ] Никогда не применяет action без явного action plan
- [ ] Покрыт unit-тестами на все 7 интентов
- [ ] Latency < 800ms

---

### M2 — Candidate Search Engine
**Приоритет:** 🔴 P0  
**Зависимости:** M1  
**Слой:** Action Layer

**Что делает:**
- Ищет кандидатов в VK по семантическим запросам (не только keywords)
- Фильтрует по гео, дате активности, типу аккаунта
- Отсекает ботов, бизнес-страницы, экспертов и конкурентов на этапе search
- Возвращает сырой пул кандидатов для Reranker

**Definition of Done:**
- [ ] Возвращает ≥ 20 кандидатов по любому запросу, связанному с астрологией / таро / нумерологией
- [ ] Не более 20% экспертов/продавцов в сыром пуле (до rerank)
- [ ] Поддерживает фильтры: город, пол, возраст, дата последней активности
- [ ] Логирует каждый поиск с параметрами в state storage

---

### M3 — Reranker
**Приоритет:** 🔴 P0  
**Зависимости:** M2  
**Слой:** Action Layer

**Что делает:**
- Принимает сырой пул из M2
- Ранжирует по "теплоте" кандидата (желание учиться, признаки запроса)
- Применяет активные positive/negative markers из Learning Layer
- Отдаёт топ-N кандидатов для финального Judge

**Модель:** Специализированный reranker-сервис (Cohere Rerank / Cross-encoder)

**Definition of Done:**
- [ ] Precision@5 ≥ 0.75 на размеченном тест-сете из 100 кандидатов
- [ ] Применяет все active rules из learning_config без исключений
- [ ] Объясняет score для каждого кандидата (список сработавших markers)
- [ ] Latency < 2s для пула из 50 кандидатов

---

### M4 — Lead Judge
**Приоритет:** 🔴 P0  
**Зависимости:** M3, M7 (Knowledge Base)  
**Слой:** Action Layer

**Что делает:**
- Финальная оценка каждого кандидата из топа Reranker
- Отвечает на вопрос: «Это ученик или нет?»
- Объясняет решение в 2-3 предложениях на русском
- Присваивает уровень доверия: HIGH / MEDIUM / LOW
- Отказывается принимать кандидата без достаточного контекста

**Модель:** Сильная модель (GPT-4o / Claude Sonnet) — здесь дорого ошибиться

**Definition of Done:**
- [ ] Accuracy ≥ 85% на размеченном тест-сете из 80 кандидатов
- [ ] Каждое решение содержит объяснение ≥ 2 предложений
- [ ] Ложноположительных экспертов/конкурентов < 10%
- [ ] Не галлюцинирует данные о кандидате — ссылается только на контекст

---

### M5 — Message Generator
**Приоритет:** 🟡 P1  
**Зависимости:** M4, M7  
**Слой:** Action Layer

**Что делает:**
- Генерирует первое сообщение от лица Дарьи конкретному кандидату
- Опирается на профиль кандидата + актуальные продукты из Knowledge Base
- Соблюдает style rules (тон, длина, без пафоса)
- Предлагает 1-2 варианта на выбор Дарьи
- Поддерживает режим редактирования: «перепиши короче», «без упоминания цены»

**Модель:** GPT-4o / Claude Sonnet

**Definition of Done:**
- [ ] Сообщение всегда соответствует активным style rules
- [ ] Не упоминает продукты или цены, которых нет в Knowledge Base
- [ ] Дарья оценивает ≥ 70% сгенерированных сообщений как «подходит» или «подходит после правки»
- [ ] Поддерживает ≥ 5 команд редактирования в одном диалоге

---

### M6 — Feedback & Learning Loop
**Приоритет:** 🟡 P1  
**Зависимости:** M4, M3  
**Слой:** Learning Layer

**Что делает:**
- Принимает метку Дарьи по кандидату: ✅ ученик / ❌ не то / ⚠️ спорно
- Накапливает историю feedback в структурированном хранилище
- Анализирует паттерны ошибок через batch-анализ (сильная модель)
- Формулирует предложения по изменению правил (rule patches)
- Показывает патчи Дарье на утверждение — **без молчаливого применения**
- Применяет утверждённые патчи в learning_config

**Модель:** Batch-анализ — GPT-4o / Claude Sonnet

**Definition of Done:**
- [ ] После 10+ feedback-меток предлагает ≥ 1 конкретный rule patch
- [ ] Ни один патч не применяется без явного «да» от Дарьи
- [ ] После применения патча точность отбора в следующих 10 кандидатах улучшается
- [ ] История feedback экспортируется в JSON с полной трассировкой

---

### M7 — Knowledge Base (darangel.ru)
**Приоритет:** 🔴 P0  
**Зависимости:** нет  
**Слой:** Knowledge Layer

**Что делает:**
- Хранит структурированный справочник курсов, продуктов и услуг Дарьи
- Первичный источник: парсинг darangel.ru + ручная валидация
- Поддерживает grounded-generation: ответы по продуктам только из KB, без галлюцинаций
- Обновляется вручную (с подтверждением) при изменении продуктовой линейки
- Включает: названия, форматы, цены (если публичны), описания, целевую аудиторию

**Definition of Done:**
- [ ] Все публичные курсы с darangel.ru занесены и валидированы
- [ ] Ответы на вопросы о продуктах содержат только KB-данные (0 галлюцинаций на тест-сете из 30 вопросов)
- [ ] KB доступна как structured JSON + vector index для retrieval
- [ ] Процедура обновления KB задокументирована и занимает ≤ 30 минут

---

### M8 — Safety & Confirmation Layer
**Приоритет:** 🔴 P0  
**Зависимости:** M1, M6  
**Слой:** Safety Layer

**Что делает:**
- Перехватывает все действия, которые изменяют правила отбора, style rules или KB
- Показывает Дарье что именно изменится (diff) и просит подтверждение
- Блокирует применение без ответа «Да» / «Подтверждаю»
- Логирует все изменения с timestamp и обоснованием

**Definition of Done:**
- [ ] 100% стратегических изменений проходят через confirmation flow
- [ ] Diff читаем: показывает «было» / «станет» в Telegram-сообщении
- [ ] Журнал изменений доступен по команде «покажи историю изменений»
- [ ] Невозможно обойти Safety Layer через любой из поддерживаемых интентов

---

### M9 — State Storage & Persistence
**Приоритет:** 🔴 P0  
**Зависимости:** нет (базовая инфраструктура)  
**Слой:** Action Layer / инфраструктура

**Что делает:**
- Хранит: learning_config, feedback history, search history, KB, style rules, positive/negative markers
- Поддерживает load/save state
- Поддерживает экспорт/импорт для резервного копирования
- Персистентность между перезапусками бота

**Definition of Done:**
- [ ] Все модули читают/пишут state через единый Storage API
- [ ] State переживает перезапуск бота без потери данных
- [ ] Резервная копия создаётся автоматически раз в сутки
- [ ] Полный state экспортируется в < 5 секунд

---

### M10 — Telegram Bot Interface
**Приоритет:** 🔴 P0  
**Зависимости:** M1, M4, M5, M6, M8  
**Слой:** Conversational Layer

**Что делает:**
- Telegram-интерфейс для Дарьи
- Карточки кандидатов с кнопками: ✅ / ❌ / ⚠️ / 💬 написать
- Отображение объяснения Judge и score Reranker
- Команды на естественном языке (не slash-команды)
- Уведомления о готовом auto-search, новых кандидатах, предложенных патчах

**Definition of Done:**
- [ ] Карточка кандидата содержит: имя, ссылка, объяснение, уровень доверия, кнопки
- [ ] Feedback-кнопки работают и сохраняются в storage
- [ ] Бот понимает свободный текст (не только /команды)
- [ ] Бот отвечает на «что ты умеешь» понятным кратким списком

## ✅ КРИТЕРИИ УСПЕХА HERMES V3

1. **Точность отбора:** ≥ 80% кандидатов из топ-5 — реальные потенциальные ученики
2. **Нет галлюцинаций:** 0 выдуманных фактов о продуктах Дарьи
3. **Объяснимость:** каждый лид сопровождается объяснением выбора
4. **Обучаемость:** после 20 меток качество отбора улучшается на ≥ 10%
5. **Безопасность:** 100% стратегических изменений требуют подтверждения
6. **Стоимость:** средняя стоимость обработки 1 кандидата < $0.02
7. **Скорость:** от запроса до карточек ≤ 15 секунд

## 🚫 ЧТО HERMES V3 НЕ ДЕЛАЕТ

- Не ищет партнёров, экспертов, коллег, конкурентов
- Не применяет изменения правил без подтверждения Дарьи
- Не фантазирует о продуктах и ценах Дарьи
- Не работает как полностью автономный агент без human-in-the-loop
- Не использует дорогие модели для тривиальных задач
